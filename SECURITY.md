# Security policy

## Supported versions

Security and physical-safety fixes are applied to the latest release and the
`main` branch.

## Reporting a vulnerability

Do not open a public issue for a vulnerability that could enable unsafe robot
motion, bypass target or limit checks, expose credentials, or compromise a
deployment host. Email `morrje@kth.se` with:

- the affected version or commit;
- a minimal reproduction;
- the expected impact; and
- any suggested mitigation.

You should receive an acknowledgement within seven days. Please allow time for
a coordinated fix before public disclosure.

The deployment software is research software and is not safety-certified. Use
an independent emergency stop and appropriate mechanical support; software
checks are not a substitute for physical safeguards.
